# Broader Admin UI (SP4) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Grow `/admin` beyond feeds — a read-only instance overview + a "who's on this instance" user list, behind a shared admin layout.

**Architecture:** Two read-only core endpoints (`GET /admin/overview`, `GET /admin/users`) backed by two new `repo` methods (`instanceStats`, `listUsers`) that query via `this.raw` (they touch the better-auth `user` table, which Kysely's DB interface doesn't model — same pattern `sweepAnonymousUsers` uses), delegated through `service`. The web `/admin` area becomes sub-routes behind a shared `+layout` that gates on `isAdmin`; the SP2 feed page moves to `/admin/feeds`. SP4 retires the dead `/admin/status` route.

**Tech Stack:** Hono/Node core (better-sqlite3 + Kysely, Node 22 native type-stripping — no build step, no TS parameter properties), SvelteKit web (Svelte 5 runes, plain scoped CSS + `--color-*` tokens), vitest.

**Spec:** `docs/superpowers/specs/2026-07-18-admin-ui-design.md` (rev 1).

## Global Constraints

- **No TypeScript parameter properties** in `core/src` (Node native type-stripping).
- **`repo.instanceStats()` + `repo.listUsers()` query via `this.raw`** (better-sqlite3 prepared statements) — they join/count the better-auth `user` table, which is NOT in Kysely's `DB` interface. Follow `sweepAnonymousUsers`'s raw style.
- **`instanceStats()` counts:** `registeredUsers` = `user WHERE isAnonymous=0 OR isAnonymous IS NULL`; `guests` = `user WHERE isAnonymous=1`; `remoteFeeds` = `users WHERE kind='remote'`; `posts` = `COUNT(*) FROM posts`. (Use `COUNT(*)`, not row materialization.)
- **`listUsers()`** returns registered locals + remote feeds, **excludes anonymous guests**; `emailVerified` is `boolean | null` (null for remote), `feedUrl` `string | null` (null for local).
- **Both new endpoints** are `authed` + `requireAdmin()` → admin 200 / non-admin registered 403 / anon session 403 / no session 401 (SP2 rev-2 semantics).
- **`/admin/status` is retired** (route deleted; its `admin.test.ts` integration tests repoint to `/admin/overview`).
- **Web layout gate:** `admin/+layout.server.ts` → `if (!me?.isAdmin) throw error(404)`. The moved feeds page drops its own per-page gate. Form actions stay protected by core (defense in depth).
- **UI (overview, users, nav):** built via the `ui-ux-pro-max:ui-ux-pro-max` skill + `design-system/textcaster/MASTER.md`; consult `svelte-skills` (`sveltekit-data-flow`, `svelte-runes`, `svelte-template-directives`). Colors from `web/src/app.css` `--color-*` — **no raw hex**. **Do NOT** use `ui-ux-pro-max:ui-styling` (Tailwind/shadcn — not this codebase). No new dependencies.
- **Shared checkout:** stage explicit paths, **never `git add -A`**. Commit trailer `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.
- Known flaky: a full `npm test -w core` may show one `ingest.test.ts > pollAll swallows an oversized feed` timeout — a load artifact, passes isolated; not a regression.

```ts
  listUsers(): Array<{ handle: string; displayName: string; kind: 'local' | 'remote'; emailVerified: boolean | null; createdAt: string; feedUrl: string | null }> {
    const rows = this.raw.prepare(
      `SELECT u.handle AS handle, u.display_name AS displayName, u.kind AS kind,
              u.created_at AS createdAt, u.feed_url AS feedUrl, au.emailVerified AS emailVerified
       FROM users u LEFT JOIN user au ON au.id = u.auth_user_id
       WHERE u.kind = 'remote'
          OR (u.kind = 'local' AND (au.isAnonymous = 0 OR au.isAnonymous IS NULL))
       ORDER BY u.created_at DESC`,
    ).all() as Array<{ handle: string; displayName: string; kind: 'local' | 'remote'; createdAt: string; feedUrl: string | null; emailVerified: number | null }>
    return rows.map((r) => ({ ...r, emailVerified: r.emailVerified === null ? null : r.emailVerified === 1 }))
  }
```

```ts
export async function getAdminOverview(f: typeof fetch): Promise<{ counts: { registeredUsers: number; guests: number; remoteFeeds: number; posts: number }; federation: { websub: string; rssCloud: boolean; pushIn: boolean; publicUrl: string | null }; mailEnabled: boolean; adminEmails: string[] }> {
```

**Interfaces:**

- [ ] **Step 1: Invoke the UI skill first** — `ui-ux-pro-max:ui-ux-pro-max` + MASTER.md + `svelte-skills`, as Task 3 Step 1. No `ui-styling`.

- [ ] **Step 2: Create the load** — `web/src/routes/admin/users/+page.server.ts`

```ts
import { authedFetch, cookieHeader } from '$lib/server/session'
import { listAdminUsers } from '$lib/api'
import type { PageServerLoad } from './$types'

export const load: PageServerLoad = async ({ fetch, url, cookies }) => {
	const f = authedFetch(fetch, url.origin, cookieHeader(cookies))
	return { users: await listAdminUsers(f) }
}
```

- [ ] **Step 3: Create the page** — `web/src/routes/admin/users/+page.svelte`

Empty state when `data.users` is `[]`. Read-only, no forms. `--color-*` tokens, MASTER.md.
